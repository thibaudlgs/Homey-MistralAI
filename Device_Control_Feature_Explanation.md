# How the Device Control Feature Works

## Overview
The **"Control Devices with Prompt"** feature (`control_devices_prompt`  flow action) allows users to use natural language commands to control their smart home devices in Homey. Instead of manually creating highly specific flow routines for every scenario, the user simply provides a prompt (like *"Turn off all the lights in the living room"* or *"Dim the bedroom light to 50%"*), and Mistral AI interprets the command to automatically perform the requested actions.

## Step-by-Step Breakdown

### 1. Retrieving Available Devices
When the flow action is triggered, the Homey app queries the system for all connected devices via `this.homey.devices.getDevices()`. It compiles a comprehensive list of every device's name and its supported capabilities (e.g., `onoff` for power, `dim` for brightness, `target_temperature` for thermostats, etc.).

### 2. Constructing the Prompt
The app builds a highly specific `system_prompt` that assigns Mistral the role of an intelligent "Homey smart home controller". This system prompt includes:
- **Strict Instructions**: The AI is commanded to reply **ONLY** in a specific JSON format, with no conversational text.
- **Context**: The compiled list of your actual available devices and their corresponding capabilities.
- **Rules**: Explicit formats to use for capability values (e.g., `true`/`false` for `onoff`, `0.0` to `1.0` for `dim`).

The user's actual text prompt is then appended as a user message.

### 3. Sending the Request to Mistral AI
This context is sent to the Mistral AI API. The `temperature` parameter is set extremely low (`0.1`), ensuring the AI's response is highly deterministic, factual, and strictly adheres to the JSON structure instead of generating creative text.

### 4. Parsing the AI's Response
Once Mistral AI replies, the application receives a raw text response. The app strips out any Markdown code wrappers (like ````json ... ````) that the model might have generated and parses the raw text into a JavaScript Object. 

The expected JSON structure generated by the AI looks like this:
```json
{
  "actions": [
    {
      "device": "Living Room Light",
      "capability": "onoff",
      "value": false
    }
  ],
  "explanation": "I have turned off the Living Room Light."
}
```

### 5. Executing the Actions
The app loops through the `actions` array determined by Mistral:
- **Device Matching**: It searches the Homey system for a device name that matches the one the AI specified (it handles exact matches or partial name overlaps).
- **Capability Verification**: It cross-checks if the matched device actually supports the requested capability (e.g., preventing the AI from attempting to "dim" a simple on/off smart plug).
- **State Update**: If the device and capability match are successful, the app calls `setCapabilityValue(capability, value)` to execute the specific change in the Homey ecosystem.

### 6. Flow Feedback
Once all operations finish executing, the flow action returns two variables back to Homey:
- `explanation`: A short textual summary populated by the AI detailing what it did. This is very useful for push notifications or text-to-speech feedback.
- `devices_controlled`: The total number of successful actions the app performed.

## Why this architecture is effective
- **Dynamic Context**: The app continuously reads your current Homey environment. If you add a new lightbulb, the AI knows about it immediately without any app code updates.
- **Strict JSON Response**: By forcing the LLM to output pure JSON data, the app natively avoids natural language parsing errors and secures structured code variables.
- **Action Reliability**: The very low AI temperature guarantees the model will act predictably rather than inventing fictional smart devices or executing unrecognized commands.
